# Project Rules

## Security
- `.env`로 시작하는 파일은 절대 읽거나 접근하지 말 것 (API 키, 시크릿 포함)

## Changelog
- 코드 수정이 발생하면 반드시 `CHANGELOG.md` 파일에 변경사항을 기록할 것
- 날짜, 수정 파일, 변경 내용을 간결하게 작성

## README
- 코드 수정으로 인해 `README.md`에 기록된 내용이 달라지거나 새로운 내용이 추가되어야 하면 자동으로 `README.md`를 업데이트할 것
- 대상: 사용법, 프로젝트 구조, 출력 포맷, 데이터 흐름 등 사용자가 참고하는 문서 내용
